# Manifest JSON Schema

The manifest file (`manifest.json`) drives the public-facing audio player page. It defines site metadata, individual mixes, and playlists.

## Top-Level Structure

```json
{
  "site": { ... },
  "mixes": [ ... ],
  "playlists": [ ... ]
}
```

## `site` Object

| Field     | Type   | Required | Description |
|-----------|--------|----------|-------------|
| `title`   | string | Yes      | Page title displayed in the header |
| `tagline` | string | No       | Subtitle text below the title |
| `accent`  | string | No       | Default accent color as hex (default: `#ff5500`) |
| `logoText`| string | No       | Small eyebrow label shown above the title (default: `Off Grid`) |

## `mixes` Array

Each entry represents a single audio track/mix.

| Field         | Type     | Required | Description |
|---------------|----------|----------|-------------|
| `id`          | string   | Yes      | Unique slug identifier (e.g. `"bloodbath"`) |
| `title`       | string   | Yes      | Display title |
| `artist`      | string   | No       | Artist name |
| `description` | string   | No       | Free-text description |
| `src`         | string   | Yes      | URL or relative path to the audio file |
| `thumb`       | string   | No       | URL or relative path to cover art image |
| `peaks`       | string   | No       | URL or relative path to peaks JSON file |
| `color`       | string   | No       | Accent color override for this mix (hex) |
| `tags`        | string[] | No       | Array of tag strings for categorization |
| `duration`    | number   | No       | Duration in seconds (from peaks data) |
| `releaseDate` | string   | No       | ISO date string (e.g. `"2024-01-15"`) |
| `tracklist`   | string   | No       | Raw tracklist text as authored (one track per line) |
| `tracks`      | object[] | No       | Parsed tracklist â€” see [`tracks`](#tracks-array) |

### `tracks` Array

Each mix may carry a parsed tracklist. The admin generates this from the raw `tracklist`
text on save; each entry is one track.

| Field     | Type   | Required | Description |
|-----------|--------|----------|-------------|
| `time`    | string | No       | Timestamp as written, e.g. `"04:32"` |
| `seconds` | number | No       | `time` parsed to seconds (for seeking); `null` if none |
| `artist`  | string | No       | Track artist |
| `title`   | string | No       | Track title |
| `url`     | string | No       | Link for the track (e.g. Bandcamp/Discogs); rendered as a clickable link |

## `playlists` Array

Each entry defines an ordered collection of mixes.

| Field     | Type     | Required | Description |
|-----------|----------|----------|-------------|
| `id`      | string   | Yes      | Unique slug identifier |
| `title`   | string   | Yes      | Display title |
| `description` | string | No    | Free-text description |
| `creator` | string   | No       | Creator/artist name |
| `thumb`   | string   | No       | URL or relative path to playlist cover art |
| `color`   | string   | No       | Accent color for the playlist player |
| `mixIds`  | string[] | Yes      | Ordered array of mix `id` values |

## Example

```json
{
  "site": {
    "title": "Off-Grid",
    "tagline": "No ads. No accounts. Just audio.",
    "accent": "#ff5500",
    "logoText": "Off Grid"
  },
  "mixes": [
    {
      "id": "sample-mix-01",
      "title": "Sample Mix 01",
      "artist": "Your Name",
      "description": "A short description of the mix",
      "src": "mixes/sample-mix-01.mp3",
      "thumb": "sample-mix-01.jpg",
      "peaks": "sample-mix-01.peaks.json",
      "color": "#ff5500",
      "tags": ["jungle", "hardcore"],
      "duration": 5355.23,
      "releaseDate": "2024-01-01",
      "tracklist": "00:00 Artist A - Opening Track https://artista.bandcamp.com/track/opening\n04:32 Artist B - Second Track",
      "tracks": [
        { "time": "00:00", "seconds": 0, "artist": "Artist A", "title": "Opening Track", "url": "https://artista.bandcamp.com/track/opening" },
        { "time": "04:32", "seconds": 272, "artist": "Artist B", "title": "Second Track", "url": "" }
      ]
    }
  ],
  "playlists": [
    {
      "id": "all-mixes",
      "title": "All Mixes",
      "creator": "Your Name",
      "color": "#ff5500",
      "mixIds": ["sample-mix-01"]
    }
  ]
}
```

## Notes

- `id` values must be unique within their array (mixes or playlists)
- `mixIds` in playlists reference mix `id` values; unresolved IDs are silently skipped
- `src`, `thumb`, and `peaks` should be full R2 URLs in production (e.g., `https://pub-xxxxx.r2.dev/audio/mix.mp3`). The admin UI auto-prepends the R2 public URL when uploading files
- `duration` is informational; the player gets actual duration from peaks data or audio metadata
- This file is generated by clicking **Publish** in the admin UI, which writes it to R2 from the D1 database
